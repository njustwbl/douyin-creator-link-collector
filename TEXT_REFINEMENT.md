# v1.7.0 文本清洗、纠错与本地 Qwen

当前界面不再提供智能推荐，所有处理模式均由用户手动选择。

- 基础清洗：统一 ASR 后处理和段落重建；
- 术语纠错：基础清洗后应用可解释的领域词典；
- 本地 Qwen3.5-4B：通过 Ollama 在本机完成保守纠错、断句和可读化；
- API 可读化整理：使用用户配置的 OpenAI 兼容 API 模型。

旧的本地 1.5B 中文纠错模型已从当前流程移除。`LocalCorrectionEngine` 类名仅为兼容内部调用，实际模型由 Ollama 管理。

## 本地模型安装

安装并启动 Ollama 后执行：

```powershell
ollama pull qwen3.5:4b
ollama pull qwen3-embedding:4b
```

在页面“文本清洗与纠错 → Ollama 本地模型设置”中保存配置并测试连接。默认地址为 `http://127.0.0.1:11434`。

## 四级文本版本

1. `raw`：ASR/OCR/文章提取原文；
2. `normalized`：确定性规则清洗结果；
3. `corrected`：术语或本地/API 模型处理结果；
4. `final`：用户确认或手动编辑后的最终稿。

导出优先级：`final > corrected > normalized > raw`。原始识别稿永远不会被覆盖。

## 四种模式

### 基础清洗

不加载大模型，处理异常空格、换行、重复行、重复标点、SenseVoice 标签及明显格式问题。

### 术语纠错

在基础清洗后，使用内置 AI 术语词典和用户自定义词表进行上下文约束替换。不会对所有别名做无条件全局替换。

自定义词表示例：

```text
Agent=ageent|A景|Aent
Token=taken|托肯
Claude Code=cloud code|克劳德 code
```

### 本地 Qwen3.5-4B

系统先执行基础清洗和术语纠错，再把长文本按段落与完整句子切块，通过 Ollama 串行调用 `qwen3.5:4b`。每块只提供少量只读上文，输出后继续经过安全校验。

默认优化：

- 关闭思考输出；
- 并发固定为 1，避免 8GB 显存峰值过高；
- 单块约 1200 字，按原文长度动态限制输出；
- 低温度生成；
- 首次校验失败后使用更严格提示重试；
- 重试仍不安全时，仅回退当前块。

### API 可读化整理

用于能力要求更高或本地模型效果不理想的文本。API 模式与本地模式互不自动切换，便于人工对比质量、速度和成本。

## 安全门禁

自动比较处理前后：

- 数字、日期、比例和版本号；
- URL、邮箱、路径和技术标识；
- 文本长度比例；
- 字符修改比例；
- 新增和删除内容。

存在风险时，结果仍会保存，但状态标记为 `review_required`，不能视为已确认最终稿。

## RAG 入库准备与问答中的本地模型

RAG 元数据抽取和知识库问答均新增“API / 本地 Qwen3.5-4B”手动选择。正文不会在两个引擎之间自动串联。知识库检索另可选择 Ollama 本地 `qwen3-embedding:4b` 建立独立索引。

建议测试时保持文档、切分参数、Top K 和评测题一致，分别建立 API 与本地 Embedding 索引，不要混用一套索引的文档向量和另一模型的查询向量。
