# Text Refinement

## 当前能力

v0.9.0 已完成从转写文本到简单纠错的可运行闭环：

1. 基础清洗；
2. AI 领域术语纠错；
3. 可选本地轻量模型纠错；
4. 可选 OpenAI 兼容 API 可读化整理；
5. 安全校验与人工最终稿。

## 四级版本

```text
raw         原始 ASR / OCR / 文章正文
normalized  规则清洗稿
corrected   术语、本地模型或 API 处理稿
final       用户人工确认稿
```

系统不会覆盖原始稿。导出优先级为：`final > corrected > normalized > raw`。

## 基础清洗

确定性处理：

- SenseVoice 控制标签和声音事件；
- 异常空格、重复换行和重复行；
- 中英文标点及段落结构；
- ASR 片段的统一句子重建。

## 术语纠错

使用可解释词典处理 AI 技术口播中的常见误识别，并支持：

```text
规范词=别名1|别名2
```

示例：

```text
Agent=ageent|Aent|A景
Token=taken|托肯
Claude Code=cloud code|克劳德 code
```

替换带有上下文约束，不把所有短词进行无条件全局替换。

## 本地模型与 API

- 本地轻量纠错：适合局部中文错字、漏字和轻微语病；
- API 可读化整理：适合碎句合并、标点段落恢复和口播稿可读化；
- 两者都禁止总结、扩写或添加事实；
- API Key 不进入日志，Windows 可使用 DPAPI 持久化。

## 安全门禁

处理前后自动比较：

- 数字、日期、比例和版本号；
- URL、邮箱、路径和英文技术标识；
- 文本长度和修改比例；
- 大幅新增、删除和语义漂移风险。

风险结果仍保存，但标记 `review_required`，需要人工确认。
