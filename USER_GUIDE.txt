CareerAgent Collector v1.7.0 快速使用

一、首次准备本地 Qwen
1. 安装并启动 Ollama。
2. 在 PowerShell 执行：
   ollama pull qwen3.5:4b
   ollama pull qwen3-embedding:4b
3. 启动 CareerAgent，进入“文本清洗与纠错”，展开“Ollama 本地模型设置”。
4. 点击“测试 Qwen3.5-4B”和“测试本地 Embedding”。

二、文本清洗与纠错
1. 双击 CareerAgent_Start.bat。
2. 采集并完成视频、图文或文章文本化。
3. 进入“文本清洗与纠错”。
4. 手动选择：基础清洗、术语纠错、本地 Qwen3.5-4B、API 可读化整理。
5. 点击“开始清洗与纠错”，检查安全提示后保存人工最终稿。

三、RAG 知识入库准备
1. 在“元数据抽取引擎”中手动选择“本地 Qwen3.5-4B”或“API 大模型”。
2. 本地模式固定串行处理；API 模式可按接口情况设置并发。
3. 两种模式不会自动串联或自动兜底，便于单独测试。

四、知识库检索
1. 选择 Embedding 提供商。
2. 使用 API 时填写对应模型和 Key。
3. 使用本地时选择“Ollama 本地向量”，模型填写 qwen3-embedding:4b，不需要 API Key。
4. 本地模型与 API 模型必须分别建立索引，不能直接共用原索引。

五、知识库问答
1. 选择已经建立完成的索引。
2. 在“回答模型”中手动选择“本地 Qwen3.5-4B”或“API 大模型”。
3. 本地模式默认采用较小来源数、8000 字符上下文和 800 Token 输出，以兼顾速度和显存。

说明：
- 原始识别稿永久保留。
- 本地 Qwen 由 Ollama 管理，不再使用旧的本地 1.5B 纠错模型。
- API Key 不会写入日志；Windows 记住 Key 时使用 DPAPI 加密。
- 当前四个环节都由用户分别选择，不会自动执行完整流水线。
